# PRD作成スキル (実験ノート検索システム v2.0)

このスキルは、実験ノート検索システムのプロダクト要求定義書(PRD)を作成するためのガイドです。

## プロジェクト固有の前提条件

### 既存のPRD
このプロジェクトには既に完成したPRDが存在します:
- **ファイルパス**: `docs/product-requirements.md`
- **優先順位**: 既存のPRDが最優先。このスキルは参考資料として使用。

### 新規PRD作成時
`docs/ideas/initial-requirements.md` が存在する場合、その内容を元にPRDを作成します。

## 出力先

作成したPRDは以下に保存してください:
```
docs/product-requirements.md
```

## プロジェクト固有の注意点

### ドメイン
- **対象**: 実験ノート検索システム
- **ユーザー**: 研究者、研究室メンバー
- **主要機能**:
  - RAG（検索拡張生成）による実験ノート検索
  - プロンプト管理
  - 新出単語抽出と正規化辞書管理
  - RAG性能評価

### 技術スタック
- **フロントエンド**: Next.js 15, React 19, TypeScript
- **バックエンド**: Python 3.12+, FastAPI, LangChain, LangGraph
- **ベクトルDB**: ChromaDB
- **デプロイ**: Vercel (フロントエンド), Google Cloud Run (バックエンド)

### 成功指標の例
- 検索精度: nDCG@10 ≥ 0.8
- 検索レスポンス時間: < 5秒
- ユーザー満足度: チームメンバーから「使いやすい」評価

## PRD作成の重要なポイント

1. **具体性と測定可能性**: すべての要件は具体的で測定可能
2. **ユーザー中心設計**: ユーザーストーリー形式で記述
3. **優先順位の明確化**: P0/P1/P2で分類
4. **SMART原則の適用**: 特にKPI定義において重要

詳細なガイドは、参照プロジェクトの `.claude/skills/prd-writing/` を参照してください。
