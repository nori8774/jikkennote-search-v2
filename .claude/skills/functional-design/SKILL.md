# 機能設計書作成スキル (実験ノート検索システム v2.0)

このスキルは、実験ノート検索システムの機能設計書を作成するためのガイドです。

## プロジェクト固有の前提条件

### 既存の機能設計書
このプロジェクトには既に完成した機能設計書が存在します:
- **ファイルパス**: `docs/functional-design.md`
- **優先順位**: 既存の機能設計書が最優先。このスキルは参考資料として使用。

### 依存ドキュメント
機能設計書は以下のドキュメントを参照して作成します:
- `docs/product-requirements.md` - PRD
- `docs/architecture.md` - アーキテクチャ設計書

## 出力先

作成した機能設計書は以下に保存してください:
```
docs/functional-design.md
```

## プロジェクト固有の注意点

### 主要コンポーネント
- **検索ワークフロー**: LangGraphによるエージェント実装
- **ノート取り込み**: 増分更新、新出単語抽出
- **正規化辞書**: 表記揺れの統一
- **プロンプト管理**: YAML形式で保存・編集
- **RAG評価**: nDCG等の評価指標

### データモデル
- 実験ノート（Markdown）
- 正規化辞書（YAML）
- プロンプト（YAML）
- 検索履歴（localStorage）
- 評価テストケース（Excel/CSV）

### 主要ユースケース
1. 実験ノート検索（目的・材料・方法を指定）
2. 新規ノート取り込み（新出単語判定含む）
3. 検索履歴の閲覧・再利用
4. RAG性能評価の実行

詳細なガイドは、参照プロジェクトの `.claude/skills/functional-design/` を参照してください。
